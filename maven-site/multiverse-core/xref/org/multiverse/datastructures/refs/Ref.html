10   * Default {@link org.multiverse.datastructures.refs.ManagedRef}. Changes on refs are atomic and consistent,
11   * but not completely because a transaction could suffer from the ABA problem between transactions.
12   * See the {@link org.multiverse.datastructures.refs.AbaRef} to solve this problem.
13   * <p/>
14   * It depends on the STM implementation if the ABA problem can occur btw. If the readset also
15   * is included in the conflict detection, then the ABA problem can't occur.
16   *
17   * @author Peter Veentjer
18   */

32       * be null.
33       *
34       * @param reference the reference to store in this Ref.
35       */
36      public Ref(E reference) {
37          this.reference = reference;
38      }
39  
40      @Override

44              retry();
45          }

48      }
