8   /***
9    * Annotation that can be added to a method and constructors to make them atomic. An atomic
10   * method supports the following properties:
11   * <ol>
12   * <li>A: failure Atomicity. All changes get in, or no changes get in</li>
13   * <li>C: Consistent. A AtomicMethod can expect to enter memory in a valid state, and is
14   * expected when it leaves the space is consistent again.</li>
15   * </li>I: Isolated. A transaction will not observe changes made others transactions
16   * running in parallel. But it is going to see the changes made by transaction that
17   * completed earlier. If a transaction doesn't see this, the system could start to suffer
18   * from the lost update problem</li>
19   * </ol>
20   * <p/>
21   * Unfortunately @Inherited doesn't work for methods, only for classes. So if you have an
22   * interface containing some methods that need to be atomic, it needs to be added to the
23   * implementation and not to the interface. In the future Multiverse may try to this
24   * inference itself.
25   * <p/>
26   * When the readonly property is set to true, the transaction should not be able to do any
27   * updates. It should be in the readonly mode.
28   * <p/>
29   * With the familyName groups of transactions can be identified that share similar paths
30   * of execution. Based on the familyname the stm could do all kinds of optimizations.
31   * Luckily using instrumentation this family name can be set (and it should be) based on
32   * the class/method-name/method-signature. So no need for users to initialize it explicitly
33   * unless they want to.
34   * <p/>
35   * With the retrycount the number of retries of the transaction can be controlled. For all
36   * kinds of reasons a transaction can fail, and these transactions can be retried because
37   * the next time they could succeed. An example of such a cause is optimistic locking the stm
38   * might use. The default number of retries is Integer.MAX_VALUE, but in the future this
39   * is likely going to change to prevent livelocking.
40   *
41   * @author Peter Veentjer
42   */
43  @Retention(RetentionPolicy.RUNTIME)
44  @Target({ElementType.METHOD, ElementType.CONSTRUCTOR})
45  public @interface AtomicMethod {
46  
47      boolean readonly() default false;
48  
49      String familyName() default "";
50  
51      int retryCount() default Integer.MAX_VALUE;
52  }
